# Asuka UX 逻辑重构（对齐 NextPlayer 思路）

## 目标
- 保持 clean-room 重写，不复用 NextPlayer 代码。
- 对齐播放器核心体验：可进入、可播放、可操作、可恢复。

## 本次补全内容

### 1) 启动入口 UX
- 桌面启动不再直接进入空播放器。
- 新增启动页按钮 `Open local video`，通过系统文档选择器选择本地视频。
- 文件管理器 `VIEW` 打开仍支持直达播放。

用户路径：
1. 桌面图标打开应用。
2. 点击 `Open local video`。
3. 选择 `video/*` 文件。
4. 进入播放页。

### 2) 播放页控制与手势 UX
- 手势层改为全时挂载，支持在控制层可见时点击空白区切换显示状态。
- 打开 Overlay（速度/音轨/字幕/比例）时自动隐藏控制层，关闭后自动恢复显示。
- 锁定状态新增显式 `Unlock` 按钮，避免“锁住后无法解锁”的死路。

### 3) 状态反馈 UX
- 新增缓冲中状态 `isBuffering`，在播放页展示 loading 指示器。
- 保留现有错误层、重试/下一集动作。

## 关键状态机（当前）

### 控制层
- `visible = true/false`
- `locked = true/false`
- 自动隐藏：播放中 3 秒后隐藏（未锁定）。

### Overlay
- `overlayType = AUDIO/SUBTITLE/SPEED/SCALE/null`
- 打开 overlay -> `controls.hide()`
- dismiss overlay -> 未锁定时 `controls.show()`

### 启动入口
- 如果 `intent.data != null`：直接跳播放页
- 否则：进入启动页并走文档选择器

## 与 NextPlayer 对齐后仍待补齐
- 双击快进/快退视觉指示动画（目前只做逻辑，缺弱视觉反馈）。
- 长按倍速临时播放的完整 UX（包括速度提示层）。
- 更细分的播放错误类型与用户引导文案。
- 播放列表入口与最近播放列表页。

## 建议测试点
- 桌面启动后是否可选择并打开本地视频。
- 控制层显示时点击空白区是否可隐藏。
- 锁定后是否总能通过 `Unlock` 解除。
- Overlay 打开/关闭时控制层行为是否一致。
- 弱网或大文件下 buffering 指示是否稳定显示。

## 新增对齐项（本轮）
- 顶部控制栏：标题两行省略、左右分区、渐变头部。
- 底部控制栏：时间切换（当前/剩余）、滚动按钮组、旋转入口。
- Overlay：
  - 竖屏底部抽屉、横屏右侧面板。
  - 进入/退出动效（fade + slide）。
  - 点击背景关闭，点击面板内容不误关闭。
- 手势反馈：
  - 双击快进/快退提示。
  - 长按临时倍速提示。
