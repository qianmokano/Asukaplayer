# Asuka Player 手动测试清单（含预期结果）

## 0. 预备条件
- 设备：Android 9+，可访问本地视频文件（file:// 或 content://）。
- 安装：`./gradlew :app:assembleDebug` 后 `adb install -r asuka/app/build/outputs/apk/debug/app-debug.apk`。
- 启动方式：从文件管理器打开一个视频文件，或从桌面图标进入后再选择文件。

## 1. 启动与基础播放
1. 打开一个本地视频文件。
   - 预期：播放器进入全屏，显示标题；开始播放或可立即点击播放。
2. 暂停/播放。
   - 预期：视频画面/进度条同步暂停与继续。
3. 快进/快退（如有按钮）。
   - 预期：进度正确改变，不卡死。

## 2. 控制栏显示逻辑
1. 播放中静置 3 秒。
   - 预期：控制栏自动隐藏。
2. 轻触屏幕。
   - 预期：控制栏显示并重新计时隐藏。
3. 锁定控制栏。
   - 预期：控制栏隐藏且手势不可调整进度/音量/亮度，显示锁定状态。

## 3. 进度手势（水平拖拽）
1. 在播放区域水平拖拽。
   - 预期：显示跳转提示，拖拽结束后进度跳到对应位置。
2. 视频时长为 0 或不可获取时拖拽。
   - 预期：不崩溃，不出现异常进度。

## 4. 音量/亮度手势（竖直拖拽）
1. 屏幕左侧竖直拖拽（亮度）。
   - 预期：亮度指示条变化并生效。
2. 屏幕右侧竖直拖拽（音量）。
   - 预期：音量指示条变化并生效。
3. 上下滑超出范围。
   - 预期：数值被合理限制（0-100%）。

## 5. 缩放/平移手势（双指）
1. 双指缩放。
   - 预期：缩放比例改变且有提示；松手后保持。
2. 缩放后拖拽平移。
   - 预期：画面平移但不超出允许范围。

## 6. 倍速/轨道/比例面板
1. 打开倍速面板并选择 1.5x。
   - 预期：播放速度改变且持久化。
2. 打开音轨/字幕面板并切换。
   - 预期：音轨或字幕切换生效，无崩溃。
3. 打开比例面板切换缩放模式。
   - 预期：画面缩放模式切换正确。

## 7. 画中画与后台
1. 进入画中画（PIP）。
   - 预期：小窗播放正常，可回到前台。
2. 选择“后台播放”。
   - 预期：播放继续，返回桌面；再次进入时状态正常。

## 8. 旋转与恢复
1. 横竖屏切换。
   - 预期：画面与控件布局正常，不闪退。
2. 退出后重新打开同一视频。
   - 预期：进度与选择（倍速/字幕/音轨/缩放）恢复。

## 9. 错误与重试
1. 用损坏视频或中断的 Uri 进行播放。
   - 预期：出现错误提示，点击重试可恢复或保持错误提示但不崩溃。

## 10. 性能与稳定性
1. 连续播放 3 个文件。
   - 预期：内存稳定，无卡顿，无崩溃。
2. 快速打开/退出 5 次。
   - 预期：无崩溃，无明显资源泄漏。

## Debug 菜单（快速触发）
入口：在顶部标题文字上“长按”。

可用动作：
- Show/Hide Controls
- Toggle Lock
- Open Speed/Audio/Subtitle/Scale Overlay
- Previous/Next
- Toggle Loop/Shuffle
- Enter PiP
- Background + Finish

建议用法：
- 测 UI 流程时，先用 Debug 菜单强制打开面板或切换状态。
- 只做人工测试，不要作为正式用户入口。
