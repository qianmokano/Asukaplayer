# 路线与待办

## 当前阶段

项目已经具备可用的本地播放主链路，当前工作重点不再是“补骨架”，而是继续提升可靠性、系统集成和发布质量。

最近已完成的基础收口：

- 播放运行时设置统一到单一 source of truth
- 播放页 UI / core 边界收紧为 `PlaybackScreenModel` / `PlaybackScreenDependencies`
- 去除 `PlaybackCoreRegistry`，明确唯一组合根
- 播放位置增加 checkpoint 与销毁前 flush
- 最近历史读写改为线程安全

## 近期优先级

### 1. 系统媒体集成

- 音频焦点
- 耳机事件
- 系统媒体控制一致性
- 通知与系统媒体面板体验继续打磨

### 2. 设备侧回归覆盖

- 真机 / 模拟器上的 `connectedAndroidTest`
- `ACTION_VIEW` + `ClipData`
- PiP / 后台播放
- 错误恢复链路

### 3. 媒体库与队列体验

- 扫描 / 排序策略继续优化
- 最近播放展示进一步打磨
- 多文件队列来源的体验补强

### 4. 发布与工程门禁

- 版本号与发布说明规范
- Lint / 测试门禁
- 更稳定的本地与 CI 验证路径

## 暂未优先

- 大规模 UI 重设计
- 非本地视频场景的复杂流媒体能力
- 脱离当前 Android + Media3 技术路线的多端迁移
